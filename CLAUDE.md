# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

CHABAR Order System — a Vietnamese-language single-page POS for a bubble tea shop. The entire app is two files:

- `index.html` (~3.2k lines) — HTML markup + all CSS + an IIFE containing the full JS app. No build step, no framework, no dependencies (only Google Fonts via CDN).
- `menu.json` — the menu (shop info, option groups, toppings, topping groups, categories of items).

## Running / developing

There is no build, lint, or test tooling. To work on the app:

```
python -m http.server
```

then open `http://localhost:8000/`. **Serving over HTTP is required** — opening `index.html` via `file://` makes `fetch('menu.json')` fail, in which case the app falls back to an **embedded copy of the menu** kept in `<script type="application/json" id="embedded-menu">` at the bottom of `index.html` (~line 2678). When you change `menu.json`, also update that embedded copy if you want the `file://` fallback to stay in sync.

State persists in `localStorage` under `chabar_pos_state_v2`. To reset: clear that key in DevTools.

## Architecture

### Code layout inside `index.html`

The JS is one IIFE divided by `// ====` banner comments. Key sections:

- **STATE** (~L1319) — single `state` object: `carts` (active), `archived` (completed/cancelled), `activeCartId`, `view` ('order' | 'orders'), `orderFilter`, `search`, `catFilter`, `mobileDetailExpanded`. `MENU` and `PRICE_UNIT` are module-level.
- **STORAGE** — `saveState()` / `loadState()`; called after every mutation.
- **MENU LOOKUPS** — `getItem`, `getTopping`, `getOptionGroup`, `getToppingGroup`, `getItemToppingSections` (dedupes toppings across multiple groups assigned to one item).
- **CART STATUS DERIVATION** (~L1459) — **status is always derived from boolean flags** (`paid`, `delivered`, `cancelled`) via `deriveStatus()`. Never store status as a field. `isCartEditable()` is true only for `'draft'`.
- **PRICE CALCULATION** (~L1488) — `calcLinePrice` applies size base price + option `priceDelta` (number, or per-size object like `{ "L": 5, "XL": 8 }`) + toppings. All numbers are stored in JSON in units of `shop.priceUnit` (default 1000) — display layer multiplies via `fmtPrice()`.
- **FLAG ACTIONS** (~L1590) — `setPaid`, `setDelivered`, `setCancelled`, `afterFlagChange`. **Auto-archive rule:** when `paid && delivered`, the cart moves from `state.carts` to `state.archived` automatically. `cancelled` also archives immediately.
- **ITEM CONFIG MODAL** — uses `modalState` (separate from app state) to stage edits before `commitModal()` writes a line to the active cart.
- **RENDER** — `renderAll` → `renderMenu` + `renderCart` (or `renderOrders`). `renderCart` branches on `isMobile()` (≤768px) — mobile shows the menu inside the cart panel and slides cart detail up as an overlay; desktop is a fixed two-column grid.
- **EVENT BINDING** (~L2365) — single `bindEvents()` wires every interaction via delegation; called once at init.
- **PRINT** — `printReceipt` and `printBarTicket` build HTML into `#print-area` then call `window.print()`. Print-only CSS in the `@media print` block hides the app and shows that area.

### menu.json schema

```
shop: { name, tagline, phone, hours, wifi, priceUnit }
optionGroups: [{ id, label, type:'single', required, default, values:[{ value, label, priceDelta? }] }]
toppings: [{ id, name, price }]
toppingGroups: [{ id, name, toppingIds:[...] }]   // groups of toppings; an item can reference multiple
categories: [{ id, name, items:[{ id, name, hot?, new?, sizes:[{size, price}], optionGroupIds, toppingGroupIds }] }]
```

`priceDelta` may be a flat number or a per-size object. An item's available toppings are the union of `toppingGroupIds`, deduped in `getItemToppingSections`.

### UI conventions worth preserving

- All user-facing strings are Vietnamese. Search is diacritic-insensitive via `removeAccents()` — keep using it for any name matching.
- Display order codes (`displayCode`, e.g. `123-AB`) are generated by `genDisplayCode()` with collision avoidance against active carts and same-day archived carts.
- Cart tabs can be **pinned** (`cart.pinned`) — pinned carts cannot be closed without unpinning first.
- `init()` runs a small backward-compat migration on load (ensures `displayCode` and the boolean flag fields exist on old persisted carts). When adding new cart fields, add a similar default in that block.
